副本收到主控命令后，会处理并将其应用到自身状态。这会像客户端发出的普通命令一样工作。关键区别在于副本不能向主控发送回应。

例如，如果主节点传播到副本：SET foo 1

副本必须更新其数据库，将 的值设置为 。foo1
与普通客户端的命令不同，副本不会回复 。+OK\r\n
测试赛
测试程序会生成一个Redis主控，并以副本形式执行你的程序，如下：

./your_program.sh --port <PORT> --replicaof "<MASTER_HOST> <MASTER_PORT>"
和之前的阶段一样，你的副本应该会与主控完成握手，并收到一个空的RDB文件。

一旦收到 RDB 文件，主控将向你的程序传播一系列写入命令：

SET foo 1 # propagated from master to replica
SET bar 2 # propagated from master to replica
SET baz 3 # propagated from master to replica
测试人员随后会向程序发出命令，检查命令是否正确处理。GET

$ redis-cli GET foo # expecting `1` back
$ redis-cli GET bar # expecting `2` back
# ... and so on
注释
传播的命令以RESP数组形式发送。因此命令将以 发送为 。SET foo 1*3\r\n$3\r\nSET\r\n$3\r\nfoo\r\n$1\r\n1\r\n
传播命令不保证一次发送一个。一个TCP段可能包含多个命令的字节。
